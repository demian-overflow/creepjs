Nightglow extensions

A fork of abrahamjuliot/creepjs with probes added for the Nightglow stealth browser. Source: gitlab.noogoo.ch/orderout/creepjs.

Why this fork exists

Upstream creepjs is excellent at "is this navigator object lying" — but it reads identity attributes from one place: window.navigator on the page. A spoof that patches only the page-side passes upstream's checks while still leaking through Worker contexts, iframes, and the request headers the server actually receives. The probes added in this fork target those secondary surfaces specifically, because that's what production detectors (Telegram, Cloudflare, Akamai-grade fingerprinters) actually use to flag inconsistent UAs.

The fork adds an openresty base image with a Lua-backed /echo endpoint so probes can compare JS-side claims against wire-level reality without needing an external service. Try /echo directly to see what the server received from your current request.

Probes added by this fork

→ Nightglow consistency probes

Reads the same identity attributes (UA, userAgentData, platform, language, languages, vendor, hardwareConcurrency, deviceMemory, webdriver, constructor.name) from five contexts:

  • page — top-level window.navigator
  • dedicated — DedicatedWorker via blob: URL
  • shared — SharedWorker via blob: URL (catches Nightglow's D9 — our shim runs user code inside DedicatedWorkerGlobalScope, so self.constructor.name leaks the real engine)
  • serviceWorker — best-effort registration (skipped on plain HTTP)
  • iframe — same-origin iframe via srcdoc
  • server — derived from /echo (Sec-CH-UA → brand list, etc.)

Each row is green if all contexts agree, red if any cell disagrees. Naive UA spoofs that only patch window.navigator light up the worker / iframe / server columns immediately. The same page also runs the modern Chrome API surface existence checks and the timer + async stack format probes (timer precision, Chrome-style "at async" frames in error stacks).

→ Nightglow shim probes (existing)

The original fork additions, focused on Nightglow's Servo-side shim correctness: SharedWorker constructor returns a working worker; the wrapper synthesizes a connect event with both e.source and e.ports[0]; SharedWorkerGlobalScope is aliased so typeof checks succeed; module-type wrappers buffer messages until import() resolves; navigator.locks.request invokes its callback; ServiceWorkerContainer.{ready, getRegistration, getRegistrations} exist; MessagePort.addEventListener('message') auto-starts the queue.

These map 1:1 to the D-numbered defects in nightglow/docs/defects/telegram.md. Each "leak" reading on this page corresponds to a specific Servo gap that breaks Telegram K's boot path.

Endpoints added

GET /echo — returns the request's method, URI, scheme, remote address, and full request headers (lower-cased keys; repeated headers as arrays) as JSON. Backed by an openresty Lua block (~15 lines, see nightglow.conf in the repo).

$ curl -s http://creepjs.orderout.svc.cluster.local/echo | jq .
{
  "method": "GET",
  "uri": "/echo",
  "scheme": "http",
  "remote_addr": "10.42.0.123",
  "server_addr": "10.42.0.251",
  "headers": {
    "host": "creepjs.orderout.svc.cluster.local",
    "user-agent": "Mozilla/5.0 ...",
    "accept-language": "en-US,en;q=0.9",
    "sec-ch-ua": "\"Chromium\";v=\"124\", ...",
    "sec-ch-ua-mobile": "?0",
    "sec-ch-ua-platform": "\"Windows\""
  }
}
